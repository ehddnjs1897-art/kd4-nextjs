# CPE (Cost per Enrollment) 자동 측정 셋업 가이드

> 한 명의 수강생을 모집하는 데 든 광고비. **광고 LP 개선 효과를 검증하는 유일한 진짜 지표**.

```
CPE = 월 광고비 ÷ 실제 결제 건수
```

---

## 왜 신청 건수가 아닌 CPE를 봐야 하나

| 신호 | 함정 |
|---|---|
| "신청 폼 50건 들어왔다!" | 그 중 결제 5명? 12명? — 결제율 다르면 의미 다름 |
| "전환율 30%로 늘었다!" | 광고 단가도 같이 올랐으면 매출 그대로 |
| **CPE 40만 → 25만 (37% 개선)** | ⭐ 진짜 매출 효과 |

→ 이번 `/join` 개선 효과 검증을 위해 **반드시** CPE를 추적해야 합니다.

---

## 자동화 수준 결정 — 옵션 B 추천

| 옵션 | 자동 정도 | 셋업 시간 | 운영 부담 | 추천 |
|---|---|---|---|:-:|
| A. 수동 (시트만) | 계산만 자동 | 30분 | 월 30분 입력 | — |
| **B. 광고비 자동 + 결제 수동** | **80% 자동** | **2~3시간** | **월 5분** | ⭐ |
| C. 풀 자동 (CRM 통합) | 100% 자동 | 며칠 | 0 | 트래픽 큰 경우 |

이 가이드는 **옵션 B** 기준입니다.

## 셋업 4단계

### Step 1. Google Sheets 템플릿 생성 (10분)

1. Google Sheets 새로 만들기 → 이름: `KD4 CPE Tracking`
2. 첫 행에 컬럼 헤더:

| 월 | 광고비_Meta | 광고비_Naver | 광고비_총합 | 신청 건수 | 결제 건수 | CPE | 신청→결제 전환율 |
|---|---|---|---|---|---|---|---|
| 2026-04 | (수동) | (수동) | =B+C | (자동) | (사장님 체크) | =D/F | =F/E |
| 2026-05 | (자동, Step 3) | (수동) | =B+C | (자동) | (사장님 체크) | =D/F | =F/E |

3. 자동 함수 입력 (B5 셀부터):
   - **D열 광고비 총합**: `=SUM(B2:C2)`
   - **G열 CPE**: `=IF(F2>0, D2/F2, "")`
   - **H열 전환율**: `=IF(E2>0, F2/E2, "")`
4. 시트 ID 확인: URL의 `https://docs.google.com/spreadsheets/d/{시트_ID}/edit`

✅ **소유자**: 사장님 본인 Google 계정

---

### Step 2. Make 시나리오 — 신청 건수 자동 기록 (30분)

**기존 Make 시나리오 활용** — 이미 `/api/notify` → Make 웹훅으로 신청 데이터 들어오는 중.

신청 시나리오에 행 추가:

```
[Webhook 트리거: /api/notify로부터]
  ↓
[기존 처리 (Solapi SMS, 구글시트 행 추가 등)]
  ↓ ★ 신규 추가
[Google Sheets 모듈: 'CPE Tracking' 시트의 현재월 행에서 신청 건수 +1 증가]
  - 시트 ID: Step 1의 ID
  - 시트명: Sheet1
  - 검색 기준: A열 = TEXT(TODAY(), "yyyy-mm")
  - 업데이트 셀: E열 = E열_현재값 + 1
```

**Make 모듈 설정 상세**:
1. Make 시나리오 편집 → 새 모듈 추가 → "Google Sheets" → "Search Rows"
2. 검색 조건: `A = {{formatDate(now; "YYYY-MM")}}`
3. 다음 모듈: "Update a Row" → E열에 `{{1.E + 1}}`

⚠️ **이 단계는 이미 Make 시나리오를 운영 중이시면 사장님 본인 또는 개발자가 GUI로 추가** (제가 코드로 셋업 못 함).

---

### Step 3. Meta 광고비 자동 추출 (1시간 — 가장 까다로움)

#### 3-1. Meta Marketing API 토큰 발급 (사장님 본인, 30분)

1. https://developers.facebook.com/ 로그인 (KD4 광고 계정 owner)
2. 좌상단 → "내 앱" → "앱 만들기" → "비즈니스" 선택
3. 앱 이름: `KD4-CPE-Tracker` → 만들기
4. 좌측 메뉴 "Marketing API" → 설정
5. 도구 → "Access Token Tool" → 토큰 생성
   - 권한: `ads_read`, `ads_management`
   - 토큰 만료: 60일 (2달마다 갱신 필요. 자동화 원하면 시스템 사용자 토큰 발급)
6. **토큰 복사 → 안전한 곳에 저장**
7. 광고 계정 ID 확인:
   - Meta Ads Manager → 우상단 계정 드롭다운 → `act_XXXXXXXXX` 형식

#### 3-2. Make 시나리오 — 매월 자동 실행 (30분)

**시나리오 명**: `KD4 - Monthly Meta Ad Spend → CPE Sheet`

```
[Schedule 트리거: 매월 1일 오전 10시]
  ↓
[HTTP 모듈: Meta Marketing API 호출]
  - URL: https://graph.facebook.com/v19.0/act_XXXXX/insights
  - Method: GET
  - Query Params:
    - access_token: {토큰}
    - fields: spend
    - time_range: {"since":"{전월 1일}","until":"{전월 말일}"}
    - level: account
  ↓
[JSON 파싱: data[0].spend]
  ↓
[Google Sheets 모듈: 'CPE Tracking' 시트에 행 추가]
  - A열 (월): {{formatDate(addMonths(now; -1); "YYYY-MM")}}
  - B열 (광고비_Meta): {{spend}}
```

**Date 표현식 (Make):**
- 전월 첫날: `{{formatDate(setDate(addMonths(now; -1); 1); "YYYY-MM-DD")}}`
- 전월 말일: `{{formatDate(addDays(setDate(now; 1); -1); "YYYY-MM-DD")}}`

#### 3-3. Naver 광고비 (선택 — Naver SearchAd 운영 중일 때)

Naver 광고비는 API가 까다롭습니다. **추천: 매월 1일 사장님이 Naver 광고 관리자에서 전월 광고비 확인 → 시트 C열에 수동 입력 (월 2분)**.

⚠️ Naver SearchAd API 토큰도 발급은 가능하지만 인증 흐름이 복잡 (NCloud 인증 등). 트래픽이 커지면 그때 자동화 검토.

---

### Step 4. 운영 가이드 (월 5분)

#### 매일 (사장님)

| 트리거 | 액션 |
|---|---|
| 카카오 상담 후 결제 확정 | 시트 F열(결제 건수) +1 |

→ 시트 셀 클릭해서 숫자 +1만. 또는 더 간단히 별도 탭에 결제 로그(이름·날짜)만 기록 후 월 1회 합산.

#### 매월 1일 (자동)

| 시점 | 일어나는 일 |
|---|---|
| 매월 1일 10:00 | Make가 Meta API 호출 → 시트 B열 자동 입력 |
| 매월 1일 (사장님 직접) | Naver 광고 관리자에서 전월 광고비 확인 → C열 수동 입력 (2분) |

#### 매월 1일 — 결과 확인 (5분)

시트 G열 CPE 추세 확인:

```
2026-04: CPE 40만원 (변경 전 베이스라인)
2026-05: CPE 25만원 (변경 후) → 37.5% 개선 ✓
2026-06: CPE 27만원 → 안정화
```

H열 전환율 (신청→결제):
```
2026-04: 16% (5/30)
2026-05: 16% (8/50) → 신청은 늘었지만 전환율 그대로
                    → 즉 LP 개선이 가격에 부합한 사람을 더 데려옴
                    → CPE 개선의 진짜 원인
```

## 4주 실험 설계 (최소 데이터)

진행 후 4주 후 평가 권장:

| 기간 | 데이터 | 비교 |
|---|---|---|
| **변경 전 4주** | 직전 한 달 광고비·신청·결제 | 베이스라인 |
| **변경 후 4주** | 5월 광고비·신청·결제 | 효과 |
| 결정 | 1) 안정화 → 그대로 2) 더 개선 → 다음 변경 3) 악화 → 롤백 검토 | |

⚠️ **주의**: 광고비·시즌·외부 요인 변동이 크면 데이터 노이즈 큼. 최소 4주, 권장 8주.
